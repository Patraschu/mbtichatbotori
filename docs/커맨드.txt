Vercel 명령어
vercel login       : Vercel CLI에 로그인합니다. (최초 1회 필요)
vercel deploy      : 현재 디렉터리의 프로젝트를 Vercel에 배포합니다. (Git 연동 시 ‘vercel’만 입력)
vercel dev         : 로컬에서 Vercel 환경과 동일하게 개발 서버를 실행합니다.
vercel logout      : Vercel CLI에서 로그아웃합니다.
vercel env pull .env.local

npm 명령어
npm init -y        : 빈 폴더에 package.json 파일을 생성합니다.
npm install <패키지> : 지정한 패키지를 프로젝트의 node_modules에 설치합니다.
npm install -g <패키지> : 지정한 패키지를 전역(Global)으로 설치합니다.
npm run dev        : package.json의 dev 스크립트를 실행합니다. (보통 next dev)
npm run build      : package.json의 build 스크립트를 실행합니다. (보통 next build)
npm start          : package.json의 start 스크립트를 실행합니다. (보통 production 서버 구동)

Git 명령어
git init           : 현재 폴더를 Git 저장소로 초기화합니다.
git clone <URL>    : 원격 저장소를 로컬에 복제(Clone)합니다.
git clone https://github.com/Patraschu/nerdopower.git
git status         : 워킹 디렉터리의 변경 사항을 확인합니다.
git add .          : 변경된 모든 파일을 스테이징(Stage)합니다.
git commit -m "메시지" : 스테이징된 변경 사항을 커밋합니다.
git push           : 로컬 커밋을 원격 저장소에 푸시합니다.
git pull origin main          : 원격 저장소의 변경 사항을 로컬로 가져와 병합합니다.
git branch         : 로컬에 존재하는 브랜치 목록을 확인합니다.
git checkout <브랜치명> : 지정한 브랜치로 이동합니다. (옵션 -b로 새 브랜치 생성)
git merge <브랜치명> : 현재 브랜치에 다른 브랜치의 변경 사항을 병합합니다.
git log            : 커밋 히스토리를 조회합니다.
